# 第 6 小时：规则系统

## 本章定位

给 Agent 增加行为约束和护栏。

这是一节面向初学者的渐进式课程。阅读时请先问自己：这一章比上一章究竟多了什么？只有把这个问题答清楚，后面的框架、协议和工程实践才不会变成一堆孤立名词。

## 先建立直觉

你可以把这一章理解成：在上一章已有能力的基础上，再给 Agent 补上一块新的能力拼图。学习重点不是把 API 背下来，而是看清这块拼图在整体系统里处于什么位置、解决什么问题。

## 对照源码应该怎么读

本章主代码在 `src/main.py`。建议按下面顺序阅读：

1. 先看文件头注释，建立全局视图。
2. 再看 `main()`，确认案例是怎么启动的。
3. 接着找主流程函数，理解这章的执行主线。
4. 最后回看辅助函数和降级分支，把细节补齐。

## 新手最容易卡住的点

- 不要只看最终输出，更要看中间的状态变化、工具调用、路由或消息累积。
- 不要把“能跑”误当成“学会”。你至少应该说得清每个关键函数为什么存在。
- 如果一时看不懂，先运行案例，再回来对照日志看代码，理解会快很多。

## 建议动手实验

- 修改默认输入，观察流程是否发生变化。
- 故意改坏一个关键函数，再运行一次，看系统会在哪一步失效。
- 对照上一章，找出本章新增的代码区域。
