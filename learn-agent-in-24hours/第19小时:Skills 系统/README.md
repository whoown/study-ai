# 第 19 小时：Skills 系统

## 本章目标

- 理解 **Skill（技能）** 在 Agent 开发里的含义：把「可复用能力」封装成带说明的模块，由调度层按需选用。
- 对比「把所有逻辑写进一个巨大 prompt」与「技能注册表 + 单 Agent 调度」的差异。
- 在 `src/main.py` 里跑通一个最小闭环：**用户问题 →（可选）模型选择技能 → 执行技能 → 汇总输出**。

## 核心概念

- **技能注册表**：用字典或列表维护 `技能名 → 元数据 + 可调用函数`，相当于给 Agent 一份「能力目录」。
- **单 Agent 调度**：本章仍只有一个「大脑」负责读入用户意图并决定调用哪个技能；变化在于能力被拆成独立单元，便于测试与扩展。
- **与多智能体的关系**：下一小时会把「不同职责」拆成多个角色；本章先练好「能力模块化」，后面再把模块分配给不同智能体。

## 案例设计

场景：用户用自然语言提问，系统优先尝试用 **OpenAI 兼容 API** 的函数调用（tools）选择技能；若未配置 `OPENAI_API_KEY`，则走 **关键词路由** 的教学模拟路径，保证你仍能观察完整流程。

预置三个技能：

| 技能 | 作用 |
|------|------|
| `get_time` | 返回当前本地时间（演示「工具型」技能） |
| `word_count` | 统计用户输入中的字数（演示「轻量处理」技能） |
| `echo_topic` | 把主题原样复述（演示「占位/衔接」技能） |

## 代码讲解

本章入口为 `src/main.py`，建议按下面顺序阅读。

- **`Skill`（数据类）**：描述技能名称、说明与执行函数，是注册表里的最小单元。
- **`SKILL_REGISTRY`**：全局注册表；新增技能时只需 `register_skill(...)`。
- **`pick_skill_with_llm`**：在配置了 API Key 时，让模型从工具列表里选出要调用的技能及参数（若调用失败会打印原因）。
- **`pick_skill_mock`**：无 Key 时的降级路径，用简单关键词规则演示「调度」概念。
- **`run_once`**：串联「选技能 → 执行 → 打印结果」，是本章的**主演示函数**。

## 运行方式

在仓库的 `learn-agent-in-24hours` 目录下（已安装 `requirements.txt` 依赖）：

```bash
cd "第19小时:Skills 系统"
python src/main.py
```

若要用真实模型调度，请先在上一级目录配置 `.env` 中的 `OPENAI_API_KEY`（以及按需修改 `OPENAI_BASE_URL`、`OPENAI_MODEL`）。未配置时会自动进入模拟模式，并在终端打印明确提示。
