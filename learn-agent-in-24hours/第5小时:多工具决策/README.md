# 第 5 小时：多工具决策

## 本章目标

- 在**工具变多**时，观察模型如何做「下一步选谁」的决策（仍用手写循环，不用编排框架）。
- 理解：工具越多，**schema 描述**与**系统提示约束**越重要。
- 继续强化 `tool_calls` 协议与 `tool` 回填。

## 核心概念

- **工具竞争**：多个函数都能“似乎相关”，需要描述清晰边界。
- **任务分解**：真实问题常被拆成 2~4 次工具调用；本小时用一个小故事把它们串起来。

## 案例设计

用户任务：  
1）用 `get_unit_price` 查询 `banana` 单价；  
2）用 `multiply` 计算 `单价 * 3`；  
3）用 `say` 输出「买 3 斤香蕉花费：<金额>」。  

无 Key 时仍用分阶段 mock，保证演示稳定。

## 代码讲解

文件：`src/main.py`。

- **`TOOL_SPECS`**：包含 `get_unit_price`、`multiply`、`say`、`finish`。
- **`get_unit_price(item)`**：教学用本地“价目表”，不是真检索。
- **`mock_choice(messages)`**：按已产生的 `tool` 消息数量推进剧情。
- **`run_session(user_goal)`**：与第 4 小时同构，但工具更多、决策更复杂。

## 运行方式

```bash
cd "第5小时:多工具决策"
python src/main.py
```
