# 《24小时学会 Agent 开发》设计说明

## 项目目标

这是一套以“每小时只多学一点”为原则设计的 Agent 教学工程。

- 学习节奏必须平滑，从最小概念一路演进到真实工程能力。
- 每一小时都同时包含理论说明与最小可运行代码。
- 教程重在建立心智模型与工程直觉，不追求覆盖所有生产细节。
- 学习者应该可以顺序阅读，也可以按小时单独运行案例。

课程总提纲见 `core/syllabus.md`。

## 总体原则

### 1. 演进粒度足够小

每一小时只新增一个核心能力，其余部分默认继承自前面小时已经掌握的内容。这样做有两个目的：

- 避免初学者在一章里同时吸收太多新概念。
- 让学习者清楚看到 Agent 是如何一步步长出来的，而不是一上来就被框架封装掩盖本质。

### 2. 理论与代码必须对应

每一小时都必须包含：

- `README.md`：详细讲解本章知识点、核心流程、关键代码片段、运行方式。
- `src/main.py`：一个最小但完整、可以直接运行的教学示例。

`README.md` 不能只讲概念，必须结合本章代码解释关键实现。

### 3. 代码以教学可读性为第一目标

- 统一采用 Python 实现。
- 所有入口统一为 `src/main.py`。
- 单文件尽量控制在 500 行以内。
- 避免拆出过多辅助文件，除非某一小时确实需要。
- 代码注释要充足，并且以“帮助学习者理解为什么这样写”为目的。

### 4. 兼容 OpenAI 风格接口

所有涉及模型调用的案例，都默认采用兼容 OpenAI 风格的 API。

- 默认配置项包括：`OPENAI_API_KEY`、`OPENAI_BASE_URL`、`OPENAI_MODEL`
- 兼容 OpenAI 官方接口，也兼容 DeepSeek 一类 OpenAI 兼容服务
- 根目录提供统一配置模板，避免每章重复配置

## 目录设计

正式目录固定为 `learn-agent-in-24hours`，不再扩散到其它同义目录。

推荐目录结构如下：

```text
learn-agent-in-24hours/
├── README.md
├── requirements.txt
├── .env.example
├── run_demo.py
├── core/
│   ├── design.md
│   └── syllabus.md
└── 第X小时-主题/
    ├── README.md
    └── src/
        └── main.py
```

说明：

- 根目录只保留一份共享 `requirements.txt`
- 整个教程推荐只使用一份虚拟环境，例如 `.venv`
- 通过统一脚本 `run_demo.py` 启动任意小时的案例

## 依赖与环境策略

建议采用“单仓共享依赖”的方式，而不是每一小时单独维护一份 `requirements.txt`。

原因如下：

- 24 个小项目分别维护依赖，学习成本和维护成本都过高。
- 课程后半段会逐步引入 LangChain、LangGraph、Chroma、FastAPI、Streamlit 等能力，共享依赖更适合教学工程。
- 学习者只需要创建一次虚拟环境、安装一次依赖，就能从第 1 小时一路跑到第 24 小时。

推荐使用方式：

1. 在根目录创建 `.venv`
2. 安装根目录 `requirements.txt`
3. 通过 `python run_demo.py 章节号` 运行任意案例

## 内容分层策略

课程按照四个阶段推进：

### 第一阶段：Agent 基础心智模型（1-8 小时）

- 最小 Agent
- 接入真实 LLM
- ReAct 循环
- Function Calling
- 多工具决策
- Rules
- 短期记忆
- 上下文窗口管理

这一阶段坚持“少框架、重原理”，主要用手写 Python 代码完成。

### 第二阶段：框架与标准协议（9-12 小时）

- LangChain
- LangGraph
- MCP
- 第一个综合实战项目

这一阶段重点讲“框架替你封装了什么”，不能只是会用，不理解底层。

### 第三阶段：检索、规划、反思与状态（13-18 小时）

- Embedding 与向量数据库
- RAG
- 主动检索 Agent
- Planning
- Reflection
- 状态持久化与恢复

这一阶段开始从 Demo 思维进入工程思维。

### 第四阶段：可组合与可上线（19-24 小时）

- Skills
- 多智能体角色分工
- 多智能体编排
- 多智能体综合实战
- Human-in-the-Loop
- API 与 UI 部署

这一阶段的重点不再是“单个 Agent 能不能跑”，而是“系统是否可控、可组合、可交互”。

## 章节编写要求

每一章的 README 推荐采用以下结构：

1. 本章目标
2. 上一章到这一章新增了什么
3. 核心概念
4. 案例设计
5. 代码讲解
6. 本章小结
7. 运行方式

每一章的代码要满足以下要求：

- 默认直接使用 `python src/main.py` 可以运行
- 即使没有配置 API Key，也应该给出明确提示或降级行为
- 输出尽量可观测，让学习者能看到中间状态，而不是只看到最终答案
- 复杂流程优先用打印日志展示思考、动作、观察和状态变化

## 教学取舍

这套课程是入门与进阶之间的桥梁，不是完整的生产实践大全，因此做以下取舍：

- 优先使用最小可理解案例，不追求完全工程化
- 必要时用“教学化模拟”代替真实外部服务，降低运行门槛
- 在后半段再逐步引入更贴近真实工程的框架与模式

## 运行体验目标

最终希望学习者获得以下体验：

- 安装一次依赖
- 配置一次 API 信息
- 用统一命令运行所有小时案例
- 边读 README，边对照源码理解 Agent 的演进过程

示例命令：

```bash
python run_demo.py 1
python run_demo.py 12
python run_demo.py --list
```
你是以为经验丰富的 AI-Agent 教学老师：

我想用一种平缓过度的方式快速学习 AI Agent 技术。我称其为《24小时学会 Agent 开发》。

- 这种学习方法采用一种演进模式，每个小时都比前一个小时多学一点东西，从简单到复杂，从而形成一个非常平滑的演进过程。这样一方面可以保证不会丢下 Agent 的核心概念和基础原理，例如只会使用框架，而不熟悉 Agent 本质。另一方面，学习的演化粒度足够小，不会有太大的心智负担，1小时的学习基本可以满足要求。
- 这份教程本质上还是 Agent 的入门学习，并不能完全替代真正的工程实践。真正在项目开发过程中遇到的大量现实问题及解决的奇思妙思，是24小时所远不够学习的。
- 我将Agent 从最初级的概念，直到目前相对最前沿的业界实践前沿，拆成24个演化环节，每1小时演进一次。24小时通关这份基础学习教程。
- 我将采用理论+实践的方式进行讲解，每个演进环节中包括 README.md 和 src 目录，前者包含这个演进环节中学习的主要内容。后者包含一份可运行的 python 代码。python 代码中具备充分的注释和讲解内容，因为这份源码就是用于教学使用的。

现在请参考 @syllabus.md 中的提纲，帮我实现24小时的全部教程，要求如下：

1. 每个小时教程都是一个单独目录，命名格式为：第{{x}}小时-{{内容简介，极短}}（目录名避免使用冒号，用连字符分隔）
2. README.md 和 src 目录。
   - 其中 README.md 是本章详细的知识讲解；讲解的过程中需要引用代码库进行解说；
